# 9 · IA de Productividad (Módulo 10)

Asistente sobre OpenAI con **function-calling** (acciones reales, no solo texto)
y **RAG** (pgvector) sobre tareas, proyectos y docs del tenant.

## 9.1 Arquitectura

```
Pregunta usuario
   │
   ▼
/ai/ask ──► Orquestador (Nest)
   │           ├─ recupera contexto: estado del proyecto (SQL) + embeddings (pgvector)
   │           ├─ arma system prompt con datos del tenant (scoped por orgId)
   │           └─ OpenAI chat.completions con tools[]
   ▼
 tool_call ──► ejecuta función real (getOverdueTasks, rescheduleTask…)
   │            (respeta RBAC del usuario)
   ▼
 respuesta + acciones sugeridas (con botón "Aplicar")
```

## 9.2 Herramientas expuestas al modelo (tools)

```ts
const tools = [
  { name: 'get_overdue', desc: 'Tareas/proyectos atrasados', args: { scope } },
  { name: 'get_project_risk', desc: 'Riesgo por proyecto', args: { projectId? } },
  { name: 'get_today_tasks', desc: 'Qué hacer hoy según prioridad y capacidad', args: { userId } },
  { name: 'find_bottlenecks', desc: 'Cuellos de botella por dependencias/capacidad' },
  { name: 'reschedule_task', desc: 'Propone nueva fecha/timeboxes', args: { taskId } },
  { name: 'plan_week', desc: 'Genera timeboxes de la semana', args: { userId } },
  { name: 'summarize', desc: 'Resume doc/acta de reunión', args: { docId|text } },
];
```

Las funciones reutilizan el dominio puro:
- `get_today_tasks` → ordena por `priorityScore` (Módulo 5) filtrando por
  capacidad disponible (Módulo 7).
- `plan_week` / `reschedule_task` → `generateTimeboxes` (Módulo 6) sobre la
  agenda real del usuario.
- `find_bottlenecks` → `computeSchedule` (Módulo 4): tareas en camino crítico con
  holgura 0 + personas en rojo.
- `get_project_risk` → fórmula de riesgo (abajo).

## 9.3 Fórmula de riesgo de proyecto (determinista, la IA la explica)

```
risk = 100 * normaliza(
    0.30 * %tareas_atrasadas
  + 0.20 * %tareas_bloqueadas
  + 0.20 * sobre_ejecución_horas (actual/estimado)
  + 0.15 * ocupación_equipo_excedida
  + 0.15 * proximidad_due_date_sin_avance
)
```
Se persiste en `Project.riskScore` y dispara estado `AT_RISK` sobre umbral.

## 9.4 Ejemplos de interacción

> **"¿Qué está atrasado?"** → llama `get_overdue` → *"3 proyectos y 12 tareas.
> El más crítico: OPS-12 (vence hoy, 0% avance). [Ver] [Reprogramar con IA]"*

> **"¿Qué debería hacer hoy?"** → `get_today_tasks` → lista ordenada por score y
> ya cabida en tu capacidad (87%); ofrece *"Planificar mi día"* (crea timeboxes).

> **"¿Qué proyecto tiene más riesgo?"** → `get_project_risk` → *"Web Corp: 78/100.
> Driver principal: 40% tareas atrasadas + Ana al 128%."*

## 9.5 Guardarraíles

- Contexto **siempre scoped por `orgId`**; el modelo nunca ve datos de otro tenant.
- Las acciones que mutan datos requieren **confirmación explícita** del usuario
  y pasan por el mismo RBAC (la IA actúa "como" el usuario, no por encima).
- Caché de respuestas por hash(prompt+contexto) para coste/latencia.
- Trazabilidad: cada acción IA → fila en `Activity` con `source: "ai"`.
- Cuota por plan (créditos), modelo por tier (clasificación = mini, planes = full).
